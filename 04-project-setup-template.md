## The 5-sentence Project setup template

Copy this into the Project Instructions box when you create a new Project. Fill in every blank — the more specific you are, the better Claude performs.

```
My name is [name] and I run [business/work type].
My audience/clients are [describe them].
My tone is always [formal/casual/warm/direct — pick one].
My main goal right now is [current priority].
Things to always remember: [any rules, preferences, or context].
```

## 3 filled-in examples

**Content creator**
```
My name is Zara and I run a lifestyle and wellness Instagram account (@zaraliving).
My audience are women aged 22–35 who follow fitness, mindfulness, and healthy eating.
My tone is always warm, relatable, and motivating — like a supportive friend.
My main goal right now is growing my account from 8k to 20k followers in 6 months.
Things to always remember: I never use the word "journey." I always end posts with a 
question. I post 5 times a week (Mon, Tue, Thu, Fri, Sun).
```

**Freelancer**
```
My name is Bilal and I'm a freelance graphic designer specialising in brand identity.
My clients are small businesses and startups, usually founders aged 28–45.
My tone is always professional but approachable — confident without being stiff.
My main goal right now is landing 2 new retainer clients this quarter.
Things to always remember: My rate is £85/hour. I only take on 3 clients at a time. 
Always mention my portfolio link (bilaldesigns.co.uk) in outreach emails.
```

**Business owner**

My name is Priya and I own a boutique skincare brand called Glow By P.

My main goal right now is launching our new Vitamin C serum in Q3.

**Student**

## Projects vs Memory — what's the difference?

This trips people up, so here's the simple version:

- **Project Instructions** = what you *tell* Claude on purpose. The 5-sentence template above. It's permanent, you control it, and it applies to every chat inside that Project.
- **Memory** = what Claude *picks up on its own* as you chat — facts it notices and saves so it doesn't forget them later.

Think of it like a new assistant: **Project Instructions** are the onboarding document you hand them on day one. **Memory** is everything they learn about you over time by working with you.

Use Projects to set the rules. Let Memory fill in the rest.

## How to create a Project in Claude Desktop

1. Open Claude Desktop and look for **Projects** in the left sidebar
2. Click **New Project** and give it a name (e.g. "My Business," "Content Work," "Client: Aisha")
3. In the **Project Instructions** box, paste your filled-in 5-sentence template above
4. Click **Save** — now start a new chat inside this Project and Claude will already know everything you told it

Every conversation inside this Project keeps that context. You never have to re-introduce yourself.

## Pro tips for getting the most out of Projects

- **Be specific, not polite.** "My tone is warm but never salesy" beats "please be nice." Claude follows concrete rules better than vague ones.
- **One Project per hat.** If you run a business *and* freelance *and* study, make three Projects. Mixed context = muddled answers.
- **Add your "never" list.** The fastest quality jump comes from telling Claude what to *avoid* (banned words, formats, claims), not just what to do.
- **Update it as you go.** Your goal in Q1 isn't your goal in Q3 — edit the instructions whenever your priority changes (see below).
- **Drop in reference files.** You can attach brand guides, past posts, or a price list to a Project so Claude matches your real work, not a generic version of it.

---

## Common mistakes to avoid

- ❌ Leaving the blanks generic ("my audience is everyone") → Claude stays generic too.
- ❌ Writing one giant paragraph → use the 5 clear lines so each instruction lands.
- ❌ Re-explaining yourself in every chat → that's the whole problem Projects solve. Set it once.
- ❌ Forgetting to start the chat *inside* the Project → instructions only apply to chats opened within it.

---

## How to update your Project later

Your context changes — so should your instructions.

1. Open the Project in the left sidebar
2. Click the Project name, then **Edit Instructions**
3. Change the line that's out of date (usually "My main goal right now…")
4. **Save** — every new chat uses the updated version immediately

A 30-second edit each month keeps Claude aligned with what you're actually working on.
